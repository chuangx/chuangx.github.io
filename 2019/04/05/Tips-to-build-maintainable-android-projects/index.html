Tips to build maintainable android projects

April 05 2019

  • Take lint warnings seriously
    • static code analysis, setup pre-commit check if possible
    • use annotations actively to facilitate lint checking
      • @Nullable, @NonNull
      • @StringRes, @DrawableRes, @ColorRes, @ColorInt
      • @MainThread, @WorkerThread
  • Strings
    • use static final strings instead of random literals everywhere
      • string var has a name, itâ€™s more explicit and gives a meaning
    • use string.xml for storing texts that are displayed to users
  • Views
    • utilize tools tag to provide proper preview in android studio
    • and of course do not use android: for preview purpose and hopefully those properties get overridden in the code.
